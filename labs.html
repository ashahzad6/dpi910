Lab 1: Getting Started with Burp Suite (12 marks)


Part 1: Learning to Burp
This activity will require you to upload a report.

Remember to include screenshots with personally identifiable information (your name).
All screenshots must be yours, and must be accompanied by explanation.

Let's start learning about Portswigger's Burp Suite.
A good place to start is at Portswigger's site. Read through this tutorial, and understand how to use Burp:
https://portswigger.net/burp/documentation/desktop/getting-started
You will see a number of links entitled Next Steps at the bottom of the tutorial, linking to the following topics:

Scan a website
Use Burp Suite for penetration testing
Working with Burp's configuration
Troubleshooting common problems

Read and understand each of these pages.
Also be sure to study the following tools linked to from "Use Burp Suite for penetration testing:

Using the Target tool
Using Burp Proxy
Using Burp Intruder
Using Burp Repeater
Scanning web sites

Check out the following video series from Hacker101 to see Burp Suite in action:

We Have a Tool, Let's Find a Target, and write a report!
Next we will set up a vulnerable web server on which to practice and show our understanding.

Choose a vulnerable app from the OWASP Vulnerable Web Applications Directory: .
https://owasp.org/www-project-vulnerable-web-applications-directory/
or the Hacker101 CTF: https://ctf.hacker101.com/
Do not use Juice Shop
You may choose either on online application, or an offline one. If choosing online, be sure to heed any restrictions (Acceptable Use Policies) that are posted.

If you do not want to be bothered searching through the Directory, please use Google's Gruyere cheesy web app, as described below:

Go to the following site and set up Gruyere for your use as described on the site.
BE SURE TO BE MINDFUL OF THE ACCEPTABLE USE POLICY FOR THIS APPLICATION!
http://google-gruyere.appspot.com/

Once you understand how to use Burp Suite and your vulnerable application, and have it up and running, try out some tools and techniques on your vulnerable site.
Remember, you are just demonstrating the use of tools for now. Hacking will come later.
I want to see you experimenting with the functionality of Burp Suite
During this course, when you learn something in the Portswigger lab, see if you can use it on your vulnerable web app.
Write a report of your activity and upload it.
